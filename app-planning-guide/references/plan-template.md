# App Plan Template

Use this template for the final response.

## 1. App Summary

- Problem statement
- Target users
- Core value proposition

## 2. Assumptions and Confirmed Facts

- Confirmed facts
- Assumptions (explicitly labeled)
- Impact of assumptions on scope/architecture

## 3. MVP Scope

- In-scope features
- Out-of-scope (Later)
- Success criteria

## 4. User Experience Design

- Primary user journey (step-by-step)
- Key screens/pages and purpose
- Error and empty-state behavior

## 5. Technical Architecture

- Recommended stack (frontend, backend, database, auth, hosting)
- High-level component breakdown
- External integrations and boundaries

## 6. Data Model (Initial)

- Core entities
- Key fields per entity
- Relationship summary

## 7. API and Backend Surface

- Core endpoints or service operations
- Auth/authorization model
- Validation and error handling approach

## 8. Non-Functional Requirements

- Security and privacy baseline
- Performance targets
- Reliability and backup strategy
- Observability (logs, metrics, alerts)

## 9. Delivery Plan

- Phase breakdown with rough sizing (`S`, `M`, `L`)
- Milestones
- Critical dependencies

## 10. Risks and Open Questions

- Top risks
- Mitigations
- Remaining open questions

## 11. Immediate Next Actions

- First 3 to 5 implementation steps
