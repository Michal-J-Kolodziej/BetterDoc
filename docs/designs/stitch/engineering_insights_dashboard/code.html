Fix for Race Condition in Redis Lock

Posted by @sarah_eng • 2h ago

#backend #redis

We encountered a subtle race condition when acquiring distributed locks. The `setnx` call wasn't atomic with the expiry setting in older versions.

// Old Implementation
if (redis.setnx(key, val) == 1) {
redis.expire(key, 10);
}
// New Implementation (Atomic)
redis.set(key, val, "NX", "EX", 10);
